4. Understanding robots.txt

robots.txtã®ç†è§£

robots.txt = A Website's Rules for Bots

The robots.txt file is a simple text file that websites use to communicate with web crawlers and bots. It tells them which parts of the site they can or cannot access.

How to Check robots.txt

Simply add /robots.txt to the end of any website's domain:

https://example.com/robots.txt

Reading robots.txt Files

âœ… Example 1: Scraping Allowed

User-agent: *
Allow: /

Interpretation: "All bots (*) can access all pages (/)"

This means scraping is generally allowed!

âŒ Example 2: Scraping NOT Allowed

User-agent: *

Interpretation: "Don't scrape anything"

This means you should NOT scrape this website.

Common robots.txt Directives

Directive Meaning Example
User-agent: * Applies to all bots The asterisk (*) means "everyone"
Allow: / Access to all pages is allowed Root (/) means entire site
Disallow: / Access to all pages is denied Don't scrape anything
Disallow: /admin/ Specific directory is off-limits Can scrape other parts, not /admin/
Allow: /jobs/ Specific directory is allowed Job listings are okay to scrape

HTML = HyperText Markup Language

HTML is the code that creates the structure and content of every web page. Think of it as the skeleton and organs of a website.

Think of HTML Like a Document Structure

HTML uses "tags" to organize content, just like a document has headings, paragraphs, and lists.

<div class="company">
  <h1>ABC Corporation</h1>
  <p class="industry">Technology</p>
  <p class="employees">500 employees</p>
  <p class="description">Leading software company...</p>
</div>

HTML Tags We Care About for Scraping

HTML Tag Purpose Example Use in Company Data
<h1>, <h2> Headings Company names, section titles
<p> Paragraphs Company descriptions, mission statements
<div> Container/Section Groups related company information together
<a> Links Links to individual company pages
<ul>, <li> Lists Lists of services, products, benefits
<table> Tables Structured data like company statistics

âœ… Good News!

You don't need to be an HTML expert. Scraping tools find and extract this HTML for you. But understanding the basics helps you identify what data you can scrape.

6. Browser Developer Tools

ãƒ–ãƒ©ã‚¦ã‚¶é–‹ç™ºè€…ãƒ„ãƒ¼ãƒ«

Developer Tools = Your Window Into Website Structure

Every modern web browser has built-in developer tools that let you see the HTML code behind any web page. This is essential for planning your web scraping.

How to Access Developer Tools

Method 1: Keyboard Shortcut

Press F12 on your keyboard

Windows/Linux: F12

Mac: Cmd + Option + I

Method 2: Right-Click

Right-click anywhere on a web page

Select "Inspect" or "Inspect Element"

What You'll See

  • Elements Tab: Shows the HTML structure of the page
  • Selector Tool: Click on it, then click on any element on the page to see its HTML code
  • Live Preview: As you hover over HTML code, the corresponding part of the page highlights
  • CSS Styles: Shows the styling applied to elements

How to Use the Element Selector

  1. Open Developer Tools (F12)
  2. Click the selector icon (usually looks like an arrow in a box) in the top-left of DevTools
  3. Hover over elements on the page - they'll highlight
  4. Click an element to see its HTML code
  5. Examine the structure to identify patterns for scraping

The Scraping Feasibility Checklist

Not all websites are good candidates for scraping. Use this checklist to evaluate potential data sources:

For Each Potential Website, Check:

  • Data Availability: Does it have the company information you need? (names, descriptions, industry, size, etc.)
  • HTML Structure: Is the data organized in a structured, consistent way?
  • robots.txt: Does the robots.txt file allow scraping?
  • Terms of Service: Are there prohibitions on scraping or data collection?
  • Ethical Assessment: Is this the right thing to do? Would scraping harm the website or users?
