This assignment focuses on building a high-quality company dataset that will serve as the foundation for your ML recommendation system. You'll design a data schema, collect structured company data, and ensure quality and consistency.

この課題は、ML推薦システムの基礎となる高品質の企業データセットの構築に焦点を当てています。データスキーマを設計し、構造化された企業データを収集し、品質と一貫性を確保します。

By completing this assignment, you will be able to:

この課題を完了することで、以下ができるようになります：

    承認されたソースから構造化された企業データを収集する

    収集されたデータを一貫した使用可能な形式で整理する
  • Perform initial quality checks on collected data
    収集されたデータの初期品質チェックを実行する
  • Document data collection methodology and decisions
    データ収集方法論と決定を文書化する

Assignment Deliverables / 課題成果物

Deliverable 1: Data Schema Document (Required)

成果物1：データスキーマ文書（必須）

Required Content / 必要な内容:

Create a comprehensive data schema document defining the structure of your company dataset.

企業データセットの構造を定義する包括的なデータスキーマ文書を作成します。

Schema Must Include:

スキーマに含める必要があるもの:

  1. Field Definitions (5-8 fields minimum)

    For each field, specify:

    • Field name (e.g., company_name, industry)
    • Data type (text, number, categorical, date, URL)
    • Description (what information this field contains)
    • Requirement level (Required, Recommended, Optional)
  2. Justification for Included Fields

    Explain why each field is important for the company recommendation project

    各フィールドが企業推薦プロジェクトにとって重要な理由を説明

  3. Standardization Rules

    Document how you'll ensure consistency (e.g., "Industry will use 5 standard categories: Technology, Healthcare, Finance, Manufacturing, Other")

    一貫性をどのように確保するかを文書化

Required Fields (Must Include):

  • Company Name
  • Industry
  • Description

Recommended Fields (Choose at least 2-5):

  • Company Size
  • Location
  • Founding Year
  • Website URL
  • Employee Count
  • Revenue
  • Culture Keywords

Format / 形式:

  • Document format: PDF or Word document | 文書形式：PDFまたはWordドキュメント
  • Length: 1-2 pages | 長さ：1-2ページ
  • Use clear headings and tables for organization | 整理のために明確な見出しと表を使用

Scoring (20 points) / 採点（20点）:

  • 5-8 fields defined with all required information: 10 points
  • Clear justification for field choices: 5 points
  • Standardization rules documented: 5 points

Deliverable 2: Company Dataset CSV File (Required)

成果物2：企業データセットCSVファイル（必須）

Required Content / 必要な内容:

Collect and organize data for 20-30 companies in CSV format.

20-30社のデータをCSV形式で収集して整理します。

Dataset Requirements:

データセット要件:

  1. Number of Companies

    Minimum: 20 companies | 最低：20社

    Target: 25-30 companies | 目標：25-30社

  2. Schema Compliance

    All columns must match your defined schema

    すべての列は定義されたスキーマと一致する必要があります

  3. Data Completeness

    Required fields: 90%+ complete

    Recommended fields: 70%+ complete

    必須フィールド：90%以上完全、推奨フィールド：70%以上完全

  4. Data Consistency

    Apply standardization rules throughout

    全体で標準化ルールを適用

Format / 形式:

  • File format: CSV (.csv) | ファイル形式：CSV
  • Encoding: UTF-8 (supports Japanese characters) | エンコーディング：UTF-8
  • First row: Column headers matching schema field names | 最初の行：スキーマフィールド名と一致する列ヘッダー
  • File naming: Week06_YourName_CompanyDataset.csv

Scoring (40 points) / 採点（40点）:

  • 20-30 companies collected: 15 points
  • High field completeness (80%+ for required fields): 10 points
  • Standardized and consistent formatting: 10 points
  • No obvious errors or duplicates: 5 points

Deliverable 3: Data Collection Documentation (Required)

成果物3：データ収集文書（必須）

Required Content / 必要な内容:

Document your data collection process, sources, and quality assessment.

データ収集プロセス、ソース、品質評価を文書化します。

Documentation Must Include:

文書に含める必要があるもの:

  1. Data Collection Strategy

    Describe your approach to collecting company data

    • Which fields did you prioritize?
    • What was your collection order?
  2. Sources Used with URLs

    List all websites used for data collection

    • Primary source(s) with URL(s)
    • Secondary sources with URL(s)
    • Which fields came from which sources
  3. Collection Methodology

    How did you collect the data?

    • Tools used (scraping tool name)
    • Process followed
    • Any challenges encountered and how you addressed them
  4. Quality Check Results

    Report completeness percentages for each field

    • Example: "company_name: 100% complete (30/30)"
    • Example: "founding_year: 80% complete (24/30)"
  5. Known Limitations and Challenges

    What issues did you encounter?

    • Missing data for certain fields
    • Inconsistent source information
    • Data that required manual research
  6. Dataset Metadata
    • Collection date
    • Total number of companies
    • Field completeness statistics
    • Sources list

Format / 形式:

  • Include in the same document as Deliverable 1 (Schema) or as separate file
  • 成果物1（スキーマ）と同じ文書に含めるか、別ファイルとして
  • Length: 2-3 pages | 長さ：2-3ページ

Scoring (20 points) / 採点（20点）:

  • Clear documentation of collection process: 6 points
  • Proper source attribution with URLs: 4 points
  • Quality check results included: 5 points
  • Known limitations acknowledged: 5 points

成果物4：AI透明性ログ（該当する場合）

Required Content / 必要な内容:

If you used external AI chatbots (ChatGPT, Claude, Gemini, etc.) during data collection or documentation, provide complete documentation.

データ収集または文書化中に外部AIチャットボットを使用した場合、完全な文書を提供します。

AI Transparency Log Must Include:

  1. All AI Prompts Used

    AIツールに送信した正確なプロンプトをコピー

    • Did you cross-reference AI-provided data with original sources?
  4. Hallucination Check

    Document any instances where AI provided false information

    • Did you verify all company data independently?
    • Did AI suggest data fields that aren't practical to collect?

  • Asking AI for suggestions on data schema design

  • Researching specific company information to fill gaps
  • Understanding data organization best practices

Common Questions / よくある質問

Q: What if I can't find 20 companies?

A: Combine multiple approved sources. Use different search terms. The minimum is 20, but try for 25-30.

Q: Is it OK if some fields are incomplete?

A: Yes. Required fields should be 90%+ complete, but optional fields can have gaps. Document what's missing.

Q: Can I collect data manually instead of scraping?

A: Yes, manual collection is acceptable, especially for supplementing missing fields. Document your methodology.

Q: How do I handle companies with Japanese names?

A: Use CSV with UTF-8 encoding. You can include both Japanese and English names if available.
