## How to Use This Template | このテンプレートの使用方法

This template helps you systematically evaluate potential data sources for your ML project. Use it to assess each data source you're considering and make informed decisions about which sources to use.

このテンプレートは、MLプロジェクトの潜在的なデータソースを体系的に評価するのに役立ちます。検討している各データソースを評価し、どのソースを使用するかについて情報に基づいた決定を下すために使用してください。

### Instructions | 手順:

1. **Fill out one evaluation** for each potential data source
   - 各潜在的なデータソースに対して1つの評価を記入
2. **Rate each criterion** honestly on a scale of 1-5
   - 各基準を1〜5のスケールで正直に評価
3. **Calculate total score** (out of 25)
   - 合計スコアを計算（25点満点）
4. **Write justification** explaining your recommendation
   - 推奨を説明する正当化を記述
5. **Compare all sources** and select the best options for your project
   - すべてのソースを比較し、プロジェクトに最適なオプションを選択

### Scoring Interpretation | スコアの解釈:

- **20-25 points**: Excellent source - highly recommended
  - 優れたソース - 強く推奨
- **15-19 points**: Good source - recommended with minor reservations
  - 良いソース - 軽微な留保付きで推奨
- **10-14 points**: Acceptable source - usable but has limitations
  - 許容できるソース - 使用可能だが制限あり
- **5-9 points**: Poor source - not recommended unless no alternatives
  - 劣るソース - 代替案がない限り非推奨
- **Below 5 points**: Unsuitable source - do not use
  - 不適切なソース - 使用しない

## Example Evaluations | 評価例

### Example 1: E-Commerce Product Data | 例1: Eコマース商品データ

**Data Source Name**: Amazon Product Reviews API
**URL**: https://api.example-amazon.com/reviews
**Type of Information**: Product reviews, ratings, customer feedback
**Data Structure**: ☑ Semi-Structured (JSON)
**Access Method**: ☑ Public API

#### Evaluation Scores:

1. **Accessibility**: 4/5 - Free API with rate limits (1000 requests/day)
2. **Data Richness**: 5/5 - Includes review text, ratings, verified purchase status, reviewer info
3. **Data Quality**: 4/5 - Generally accurate, some spam reviews present
4. **Relevance to Project**: 5/5 - Perfect for sentiment analysis project
5. **Feasibility**: 4/5 - Well-documented API, Python library available

**TOTAL: 22/25**

**Recommendation**: ☑ Use

**Justification**:
This is an excellent data source for our sentiment analysis project. The API provides rich, high-quality review data with good accessibility through a free tier. While some spam reviews exist, the volume and quality of legitimate reviews far outweigh this concern. The well-documented API and available Python libraries make implementation straightforward. The rate limit of 1000 requests per day is sufficient for our project scope. This source is highly recommended.

このデータソースは、感情分析プロジェクトに優れています。APIは無料層を通じて良好なアクセス性を持つ豊富で高品質なレビューデータを提供します。いくつかのスパムレビューが存在しますが、正当なレビューの量と品質がこの懸念を大きく上回ります。よく文書化されたAPIと利用可能なPythonライブラリにより、実装が簡単になります。1日あたり1000リクエストのレート制限は、プロジェクトの範囲に十分です。このソースは強く推奨されます。

### Example 2: Social Media Data | 例2: ソーシャルメディアデータ

**Data Source Name**: Twitter/X Academic API
**URL**: https://developer.twitter.com/en/products/twitter-api/academic-research
**Type of Information**: Tweets, user profiles, engagement metrics
**Data Structure**: ☑ Semi-Structured (JSON)
**Access Method**: ☑ Public API

#### Evaluation Scores:

1. **Accessibility**: 2/5 - Requires academic research application and approval
2. **Data Richness**: 5/5 - Comprehensive data including full archive access
3. **Data Quality**: 3/5 - High volume but includes bots, spam, and noise
4. **Relevance to Project**: 4/5 - Good for trend analysis, but may have too much noise
5. **Feasibility**: 2/5 - Complex approval process, may take weeks to get access

**Recommendation**: ☐ Maybe Use

**Justification**:
While Twitter's Academic API provides very rich data, the accessibility barriers are significant. The approval process can take 2-3 weeks and requires institutional affiliation. Data quality concerns due to bots and spam will require substantial preprocessing. However, if approved, this would be an excellent source for trend analysis. Consider this as a primary source only if you can secure access early in the project timeline, otherwise use as a supplementary source to validate findings from more accessible data.

TwitterのAcademic APIは非常に豊富なデータを提供しますが、アクセス性の障壁が大きいです。承認プロセスには2〜3週間かかり、機関への所属が必要です。ボットとスパムによるデータ品質の懸念は、大幅な前処理を必要とします。ただし、承認されれば、トレンド分析の優れたソースになります。プロジェクトタイムラインの早い段階でアクセスを確保できる場合にのみ、これを主要なソースとして検討してください。そうでなければ、よりアクセスしやすいデータから得られた発見を検証するための補足的なソースとして使用してください。

## Evaluation Tips | 評価のヒント

### Be Realistic About Accessibility | アクセス性について現実的に考える

- Test actual access before rating highly
  - 高評価する前に実際のアクセスをテスト
- Consider authentication requirements, API keys, approvals
  - 認証要件、APIキー、承認を考慮
- Check if there are rate limits or usage restrictions
  - レート制限や使用制限があるか確認

### Assess Data Richness Thoroughly | データの豊富さを徹底的に評価

- Look at sample data before committing
  - コミットする前にサンプルデータを確認
- Count the number of useful fields/features
  - 有用なフィールド/特徴量の数を数える
- Check if temporal data is available (historical records)
  - 時系列データが利用可能か確認（履歴記録）

### Verify Data Quality | データ品質を検証

- Download a sample and inspect manually
  - サンプルをダウンロードして手動で検査
- Check for missing values, duplicates, inconsistencies
  - 欠損値、重複、不整合を確認
- Look for data documentation or quality reports
  - データドキュメントや品質レポートを探す

### Consider Project Relevance | プロジェクト関連性を考慮

- Does it answer your research question?
  - 研究課題に答えるか？
- Does it cover the right time period/geography/domain?
  - 適切な期間/地理/ドメインをカバーしているか？
- Is the granularity appropriate (daily vs. hourly data)?
  - 粒度は適切か（日次対時間次データ）？

### Evaluate Feasibility Honestly | 実現可能性を正直に評価

- Do you have the technical skills needed?
  - 必要な技術スキルを持っているか？
- Can you access it within the project timeline?
  - プロジェクトタイムライン内にアクセスできるか？
- Are there legal/ethical concerns?
  - 法的/倫理的懸念があるか？
- Do you have the computational resources?
  - 計算リソースを持っているか？

## Common Pitfalls to Avoid | 避けるべき一般的な落とし穴

1. **Overestimating accessibility** - Always test access before committing
   - アクセス性を過大評価 - コミットする前に常にアクセスをテスト
2. **Ignoring legal issues** - Check terms of service and copyright
   - 法的問題を無視 - 利用規約と著作権を確認
3. **Underestimating preprocessing time** - Data cleaning takes longer than expected
   - 前処理時間を過小評価 - データクリーニングは予想より時間がかかる
4. **Falling in love with data** - Don't use a source just because it's interesting
   - データに恋をする - 興味深いという理由だけでソースを使用しない
5. **Not having a backup** - Always identify 2-3 alternative sources
   - バックアップがない - 常に2〜3の代替ソースを特定

## Next Steps After Evaluation | 評価後の次のステップ

1. **Select your top 2-3 sources** based on total scores
   - 合計スコアに基づいて上位2〜3のソースを選択
2. **Test data collection** from each source
   - 各ソースからのデータ収集をテスト
3. **Collect small samples** (100-1000 records) for exploration
   - 探索用の小さなサンプル（100〜1000レコード）を収集
4. **Document your findings** in your assignment deliverable
   - 課題成果物に発見を文書化
5. **Proceed with full data collection** from approved sources
   - 承認されたソースから完全なデータ収集を進める
