Introduction to No-Code ML Platforms

What are No-Code ML Platforms?

No-code machine learning platforms allow users to build, train, and deploy ML models without writing any programming code.

Instead of coding, users interact with visual interfaces, upload data through simple forms, and configure models by clicking buttons and selecting options.

ノーコードMLプラットフォームとは？

ノーコード機械学習プラットフォームは、プログラミングコードを書かずにMLモデルを構築、訓練、デプロイできるようにします。

コーディングの代わりに、ユーザーは視覚的なインターフェースとやり取りし、シンプルなフォームを通じてデータをアップロードし、ボタンをクリックしてオプションを選択することでモデルを設定します。

Why Use No-Code Platforms?

1. Accessibility / アクセシビリティ

No programming knowledge required. Anyone can build ML models, regardless of coding experience.

プログラミング知識は不要です。コーディング経験に関係なく、誰でもMLモデルを構築できます。

2. Speed / スピード

Train models in minutes instead of hours. No time spent debugging code or configuring environments.

数時間ではなく数分でモデルを訓練します。コードのデバッグや環境の設定に時間を費やしません。

3. Focus on Concepts / コンセプトに集中

Learn ML principles without getting distracted by syntax errors and programming challenges.

構文エラーやプログラミングの課題に気を取られることなく、ML原則を学習します。

4. Professional Algorithms / プロフェッショナルアルゴリズム

No-code platforms use the same proven ML algorithms as code-based implementations. The quality is the same.

ノーコードプラットフォームは、コードベースの実装と同じ実証済みのMLアルゴリズムを使用します。品質は同じです。

The Model Training Process

Understanding what happens during model training helps you make better decisions when configuring your ML system.

モデル訓練中に何が起こるかを理解することで、MLシステムを設定する際により良い決定を下すことができます。

Understanding Model Algorithms

Logistic Regression / ロジスティック回帰

How it works:

Finds a mathematical formula that best separates your two classes (Interested vs Not Interested). Simple and fast.

Best for:

  • Small datasets
  • Linear relationships
  • When you need speed

どのように機能するか：

2つのクラス（興味ありvs興味なし）を最もよく分離する数式を見つけます。シンプルで高速です。

最適な用途：

  • 小さなデータセット
  • 線形関係
  • スピードが必要な場合

Decision Trees / 決定木

How it works:

Creates a tree of yes/no questions about features. "Is industry = Tech? → Yes → Is size > 100? → Yes → Interested"

Best for:

  • Interpretability (easy to understand)
  • Non-linear patterns
  • Mixed data types

どのように機能するか：

特徴量に関するはい/いいえの質問の木を作成します。「業界=テクノロジー？→はい→規模>100？→はい→興味あり」

最適な用途：

  • 解釈可能性（理解しやすい）
  • 非線形パターン
  • 混合データ型

Random Forest / ランダムフォレスト

How it works:

Creates many decision trees and combines their predictions. Like asking multiple experts and taking a vote.

Best for:

  • High accuracy
  • Complex patterns
  • Reducing overfitting

どのように機能するか：

多くの決定木を作成し、それらの予測を組み合わせます。複数の専門家に尋ねて投票するようなものです。

最適な用途：

  • 高精度
  • 複雑なパターン
  • 過学習の軽減

Understanding Training Metrics

Accuracy - The Primary Metric

Accuracy = Number of Correct Predictions / Total Number of Predictions

精度 = 正しい予測の数 / 予測の総数

Example / 例: Total companies: 25 Test set (20%): 5 companies 総企業数：25 テストセット（20%）：5社 Model predictions on test set: テストセットでのモデル予測： • Company A: Predicted = 1, Actual = 1 ✓ CORRECT • Company B: Predicted = 0, Actual = 0 ✓ CORRECT • Company C: Predicted = 1, Actual = 0 ✗ WRONG • Company D: Predicted = 1, Actual = 1 ✓ CORRECT • Company E: Predicted = 0, Actual = 1 ✗ WRONG Accuracy = 3 correct / 5 total = 60% 精度 = 3正しい / 5合計 = 60%

What's a Good Accuracy?

Accuracy Range Interpretation
~50% Random guessing (no better than flipping a coin)
60-70% Model is learning! Better than random
70-80% Good performance for small datasets
80-90% Very good performance
90%+ Excellent (but check for overfitting)

Context Matters

With ~25 companies in your dataset, 60-80% accuracy is realistic and good!

データセットに約25社の場合、60-80%の精度は現実的で良好です！

Why not higher?

  • Small dataset: Limited examples for the model to learn from
  • Complex task: Human career preferences are nuanced and sometimes inconsistent
  • Feature limitations: Some important factors might not be captured in your features

What to Expect During Training

The Training Timeline

  1. Data Upload (1-2 minutes)
    Platform reads your CSV and validates format
  2. Configuration (2-3 minutes)
    You specify target variable and features
  3. Training Execution (5-15 minutes)
    Platform tries algorithms and trains model
    You'll see progress indicators
  4. Results Display (instant)
    Platform shows accuracy and performance metrics

What You'll See

  • Progress bar: Shows training completion percentage
  • Status messages: "Training model...", "Evaluating performance..."
  • Algorithm selection: Platform may show which algorithm it chose
  • Training metrics: Accuracy, precision, recall displayed
  • Feature importance: Which features mattered most (if shown)

Common Training Issues and Solutions

Issue 1: Platform won't accept CSV

Possible causes:

  • File format not pure CSV
  • Special characters in column names
  • File size too large/small

Solutions:

  • Open in Excel/Sheets, re-save as CSV
  • Rename columns (no spaces or special chars)
  • Check platform file size requirements

考えられる原因：

  • ファイル形式が純粋なCSVではない
  • 列名に特殊文字がある
  • ファイルサイズが大きすぎる/小さすぎる

解決策：

  • Excel/Sheetsで開き、CSVとして再保存
  • 列名を変更（スペースや特殊文字なし）
  • プラットフォームのファイルサイズ要件を確認

Issue 2: Training fails or returns error

Possible causes:

  • Missing values in features
  • Wrong data types
  • Target column not properly set

Solutions:

  • Check all feature columns are numeric
  • Verify label column has only 0 and 1
  • Review configuration settings

考えられる原因：

  • 特徴量の欠損値
  • 間違ったデータ型
  • ターゲット列が適切に設定されていない

解決策：

  • すべての特徴量列が数値であることを確認
  • ラベル列が0と1のみを持つことを確認
  • 設定を確認

Issue 3: Training takes very long

Normal or problem?

For ~25 companies, training should take 5-15 minutes maximum. If longer:

  • Check internet connection
  • Platform may be busy (try off-peak hours)
  • Consider different platform

正常か問題か？

約25社の場合、訓練は最大5-15分かかるはずです。より長い場合：

  • インターネット接続を確認
  • プラットフォームが混雑している可能性（オフピーク時間を試す）
  • 別のプラットフォームを検討
