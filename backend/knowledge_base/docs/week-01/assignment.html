Assignment Overview / 課題概要

Course: Machine Learning and Intelligence | 機械学習と知能

Week: 1 | 第1週

Due Date: End of Week 1 class session | 第1週授業終了時

Weight: Part of 40% Weekly Activities grade | 週次アクティビティ評価40%の一部

Submission: Through ML-101 Bot Portal | ML-101ボットポータル経由

This assignment introduces you to machine learning concepts through hands-on discovery of ML applications in your daily life. You'll work with the ML-101 Bot to identify, categorize, and analyze real-world ML systems.

この課題は、日常生活でのMLアプリケーションのハンズオン発見を通じて機械学習のコンセプトを紹介します。ML-101ボットと協力して、実世界のMLシステムを特定、分類、分析します。

Learning Objectives / 学習目標

By completing this assignment, you will be able to:

この課題を完了することで、以下ができるようになります：

  • ✅ Identify machine learning applications in everyday technology
    日常的な技術における機械学習アプリケーションを特定する
  • ✅ Distinguish between ML and traditional rule-based systems
    MLと従来のルールベースシステムを区別する
  • ✅ Categorize ML tasks by type (supervised, unsupervised, reinforcement)
    MLタスクをタイプ別に分類する（教師あり、教師なし、強化学習）
  • ✅ Connect ML concepts to the course project
    MLコンセプトをコースプロジェクトに関連付ける
  • ✅ Use AI chatbots effectively for learning
    学習のためにAIチャットボットを効果的に使用する

Assignment Deliverables / 課題成果物

Deliverable 1: ML Applications Discovery Document

成果物1：MLアプリケーション発見文書

Required Content / 必要な内容:

Identify and document 5-7 real-world ML applications that you use or encounter regularly.

あなたが定期的に使用または遭遇する5-7の実世界MLアプリケーションを特定し文書化します。

For each application, include:

各アプリケーションについて、以下を含めます：

  1. Application Name / アプリケーション名

    Example: "Netflix Movie Recommendations" | 例：「Netflix映画推薦」

  2. Brief Description / 簡単な説明

    What does it do? (2-3 sentences) | 何をするか？（2-3文）

    Example: "Netflix recommends movies and shows based on my viewing history. It learns what I like to watch and suggests similar content."

  3. ML Task Type / MLタスクタイプ

    Classification, Recommendation, Prediction, Clustering, or Other

    分類、推薦、予測、クラスタリング、またはその他

  4. Why is this ML (not traditional programming)? / なぜこれはML（従来のプログラミングではない）か？

    1-2 sentences explaining the learning component | 学習コンポーネントを説明する1-2文

    Example: "Netflix learns from millions of users' watching patterns. The system adapts to my preferences over time without being explicitly programmed."

Format / 形式:

  • Document format: PDF, Word, or Google Doc | 文書形式：PDF、Word、またはGoogle Doc
  • Length: 2-3 pages (1-2 pages minimum) | 長さ：2-3ページ（最低1-2ページ）
  • Use clear headings for each application | 各アプリケーションに明確な見出しを使用

Scoring (40 points) / 採点（40点）:

  • 5-7 applications identified: 20 points (4 points each for 5 minimum)
  • Clear descriptions: 10 points
  • Correct ML task categorization: 5 points
  • ML vs traditional programming explanations: 5 points

Example Submission Excerpt / 提出例の抜粋

Application Example / アプリケーション例

Application Name: Spotify Discover Weekly Playlist

アプリケーション名: Spotify Discover Weeklyプレイリスト

Description: Spotify's Discover Weekly is a personalized playlist that updates every Monday with 30 song recommendations. The system analyzes my listening history, including which songs I play repeatedly, skip, or save. It also looks at what similar users listen to and finds patterns in music characteristics like tempo, genre, and mood.

説明: SpotifyのDiscover Weeklyは、毎週月曜日に30曲の推薦で更新されるパーソナライズされたプレイリストです。システムは、繰り返し再生する曲、スキップする曲、保存する曲を含む私の聴取履歴を分析します。また、類似したユーザーが聴いているものを見て、テンポ、ジャンル、ムードなどの音楽特性のパターンを見つけます。

ML Task Type: Recommendation System (Collaborative Filtering + Content-Based)

MLタスクタイプ: 推薦システム（協調フィルタリング + コンテンツベース）

Why is this ML? Spotify learns my music preferences from data rather than following pre-programmed rules. The system adapts over time—if I start listening to more jazz, my recommendations shift accordingly. No programmer explicitly told Spotify "recommend jazz to this user"—it learned this pattern from my behavior.

なぜこれはMLか? Spotifyは事前にプログラムされたルールに従うのではなく、データから私の音楽の好みを学習します。システムは時間とともに適応します—もっとジャズを聴き始めると、推薦もそれに応じて変わります。プログラマーがSpotifyに明示的に「このユーザーにジャズを推薦する」と指示したわけではありません—私の行動からこのパターンを学習しました。

Tips for Success / 成功のためのヒント

Getting Started / 始め方

  • ✅ Start with familiar apps | 使い慣れたアプリから始める
    Think about apps you use daily (social media, music, shopping) | 毎日使用するアプリについて考える（ソーシャルメディア、音楽、ショッピング）
  • ✅ Use the bot as a guide | ボットをガイドとして使用する
    The ML-101 Bot will ask helpful questions | ML-101ボットが役立つ質問をします
  • ✅ Be specific in your descriptions | 説明において具体的であること
    Explain HOW the ML works, not just WHAT it does | MLが何をするかだけでなく、どのように機能するかを説明する

Common Mistakes to Avoid / 避けるべきよくある間違い

  • ❌ Confusing ML with traditional programming
    A calculator is NOT ML (it follows fixed rules). A spam filter IS ML (it learns from examples).
  • ❌ Too brief descriptions
    "Netflix uses ML" is not enough. Explain HOW and WHY it's ML.
  • ❌ Not connecting to course project
    Your reflection should explicitly mention the company recommendation system.

Frequently Asked Questions / よくある質問

Q: Can I include the same apps everyone else uses (Netflix, Spotify)?

Q: 他の人が使うのと同じアプリ（Netflix、Spotify）を含めてもいいですか？

A: Yes! It's fine to include common apps. What matters is YOUR explanation of how they use ML.

A: はい！一般的なアプリを含めても構いません。重要なのは、それらがどのようにMLを使用するかのあなた自身の説明です。

Q: What if I can't think of 7 applications?

Q: 7つのアプリケーションが思いつかない場合は？

A: 5 is the minimum requirement. Use the bot to help brainstorm—it will ask guiding questions.

A: 5つが最低要件です。ブレインストーミングを助けるためにボットを使用してください—ガイド質問をします。

Q: Can I work with classmates?

Q: クラスメートと一緒に作業できますか？

A: You can discuss ideas, but your submission must be your own work with your own examples and writing.

A: アイデアを議論することはできますが、提出物はあなた自身の作業であり、あなた自身の例と執筆である必要があります。

Q: What if the bot gives me wrong information?

Q: ボットが間違った情報を与えた場合は？

A: Use critical thinking! If something seems wrong, verify it or ask your instructor. The bot is a tool, not the final authority.

A: 批判的思考を使用してください！何かが間違っているように見える場合は、それを検証するか、インストラクターに尋ねてください。ボットはツールであり、最終的な権威ではありません。
