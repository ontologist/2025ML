Assignment Overview / 課題概要

Course: Machine Learning and Intelligence | 機械学習と知能

Week: 12 | 第12週

Due Date: End of Week 12 class session | 第12週授業終了時

Weight: Part of 40% Weekly Activities grade | 週次アクティビティ評価40%の一部

Submission: Through ML-101 Bot Portal | ML-101ボットポータル経由

Duration: 40-60 minutes | 40-60分

This assignment evaluates your trained model from Week 11 on NEW test data. You'll calculate evaluation metrics, interpret results, and critically analyze your model's performance and limitations.

この課題は、新しいテストデータで第11週の訓練済みモデルを評価します。評価メトリクスを計算し、結果を解釈し、モデルのパフォーマンスと限界を批判的に分析します。

Learning Objectives / 学習目標

By completing this assignment, you will be able to:

この課題を完了することで、以下ができるようになります：

  • ✅ Test ML models on unseen data and generate predictions
    未見のデータでMLモデルをテストし予測を生成する
  • ✅ Calculate and interpret evaluation metrics (accuracy, confusion matrix, precision, recall)
    評価メトリクス（精度、混同行列、精密度、再現率）を計算し解釈する
  • ✅ Analyze model errors and identify patterns in failures
    モデルエラーを分析し失敗のパターンを特定する
  • ✅ Think critically about AI capabilities, limitations, and ethics
    AI能力、限界、倫理について批判的に考える
  • ✅ Apply evaluation insights to improve ML systems
    評価の洞察を適用してMLシステムを改善する

Assignment Deliverables / 課題成果物

Deliverable 1: Test Dataset

成果物1：テストデータセット

Required Content / 必要な内容:

Collect and document 5-10 NEW companies that your model has NEVER seen before.

モデルが以前に見たことがない5-10の新しい企業を収集し文書化します。

Requirements / 要件:

  • Companies must be DIFFERENT from your Week 6-11 training data
    企業は第6-11週の訓練データとは異なる必要があります
  • Same data fields as training data (company name, industry, location, description)
    訓練データと同じデータフィールド（企業名、業界、場所、説明）
  • Same feature engineering applied (keywords, one-hot encoding)
    同じ特徴量エンジニアリングを適用（キーワード、ワンホットエンコーディング）
  • Include a mix: some you'd expect to be "Interested" and some "Not Interested"
    混合を含める：「興味あり」と予想されるものと「興味なし」のもの
  • Document YOUR true labels (what you're actually interested in) for later comparison
    後の比較のためにあなたの真のラベル（実際に興味があるもの）を文書化

Format / 形式:

  • File format: CSV file | ファイル形式：CSVファイル
  • File name: Week12_[YourName]_TestData.csv
  • Must include: All feature columns matching training data | 訓練データと一致するすべての特徴量列を含める必要があります

Scoring (20 points) / 採点（20点）:

  • 5-10 NEW companies collected: 10 points
  • Correct feature structure: 5 points
  • True labels documented: 5 points

Deliverable 2: Model Predictions and Results

成果物2：モデル予測と結果

Required Content / 必要な内容:

Use your Week 11 trained model to make predictions on all test companies and document results.

第11週の訓練済みモデルを使用してすべてのテスト企業で予測を行い、結果を文書化します。

Include / 含める:

  1. Prediction Table / 予測表

    For each test company, show:

    • Company name | 企業名
    • Model prediction (0 = Not Interested, 1 = Interested)
      モデル予測（0 = 興味なし、1 = 興味あり）
    • Confidence/probability (if available) | 信頼度/確率（利用可能な場合）
    • Your true label | あなたの真のラベル
    • Correct or Incorrect? | 正しいか間違っているか？
  2. Screenshots / スクリーンショット

    Include screenshots of prediction results from your ML platform

    MLプラットフォームからの予測結果のスクリーンショットを含める

Format / 形式:

  • Include in main report document (PDF or Word) | メインレポート文書（PDFまたはWord）に含める
  • Clear table format | 明確な表形式

Scoring (20 points) / 採点（20点）:

  • Predictions for all test companies: 10 points
  • Clear comparison table: 5 points
  • Supporting screenshots: 5 points

Deliverable 3: Evaluation Metrics Report

成果物3：評価メトリクスレポート

Required Content / 必要な内容:

Calculate and document all evaluation metrics with clear explanations.

すべての評価メトリクスを計算し、明確な説明とともに文書化します。

Must Include / 含める必要があります:

1. Accuracy / 精度

  • Formula used | 使用した公式
  • Calculation with numbers | 数字での計算
  • Final accuracy percentage | 最終精度パーセンテージ
  • Brief interpretation (Is this good? Why or why not?) | 簡単な解釈（これは良いか？なぜか？）

2. Confusion Matrix / 混同行列

  • Complete 2x2 matrix with counts | カウント付きの完全な2x2行列
  • True Positives (TP), True Negatives (TN), False Positives (FP), False Negatives (FN)
  • Visual representation (table or diagram) | 視覚的表現（表または図）

3. Precision / 精密度

  • Formula and calculation | 公式と計算
  • Interpretation: "Of companies model recommended, X% were actually good matches"
    解釈：「モデルが推薦した企業のうち、X%が実際に良いマッチでした」

4. Recall / 再現率

  • Formula and calculation | 公式と計算
  • Interpretation: "Of companies I'm interested in, model found X%"
    解釈：「興味のある企業のうち、モデルはX%を見つけました」

5. Visualizations (Optional but recommended) / 視覚化（オプションだが推奨）

  • Charts or graphs showing metrics | メトリクスを示すチャートまたはグラフ
  • Confusion matrix visualization | 混同行列の視覚化

Format / 形式:

  • Section in main report document | メインレポート文書のセクション
  • Clear headings for each metric | 各メトリクスの明確な見出し
  • Show your work (calculations) | 作業を示す（計算）

Scoring (30 points) / 採点（30点）:

  • Accuracy correctly calculated: 5 points
  • Confusion matrix complete: 10 points
  • Precision and recall calculated: 10 points
  • Clear interpretations: 5 points

Deliverable 4: Critical Analysis Report

成果物4：批判的分析レポート

Required Content / 必要な内容:

Write 2-3 pages critically analyzing your model's performance, errors, and implications.

モデルのパフォーマンス、エラー、影響を批判的に分析する2-3ページを書きます。

Must Address / 扱う必要があります:

Section 1: Error Analysis / エラー分析 (1 page)

  • Which specific companies were misclassified? | どの特定の企業が誤分類されたか？
  • Why do you think the model failed on these cases? | なぜモデルはこれらのケースで失敗したと思いますか？
    • Missing features? | 特徴量の欠落？
    • Unclear patterns? | 不明確なパターン？
    • Borderline cases? | 境界線ケース？
  • Are errors systematic or random? | エラーは体系的かランダムか？
  • Examples of specific failures with explanations | 説明付きの具体的な失敗の例

Section 2: Model Strengths and Capabilities / モデルの強みと能力 (0.5 page)

  • What patterns did the model learn correctly? | モデルは何のパターンを正しく学習したか？
  • Which features seem most important? | どの特徴量が最も重要に見えるか？
  • What does the model do well? | モデルは何をうまくやるか？

Section 3: Limitations and Ethics / 限界と倫理 (0.5-1 page)

  • Model Limitations: What can't this model do? | このモデルは何ができないか？
  • Bias: Did training labels introduce any bias? | 訓練ラベルはバイアスを導入したか？
  • Trust: Would you use this for real job search decisions? Why or why not? | 実際の就職活動の決定にこれを使用しますか？なぜか、またはなぜしないのか？
  • Ethics: If this was used for automated hiring, what ethical concerns arise? | これが自動採用に使用された場合、どのような倫理的懸念が生じるか？

Section 4: Improvement Suggestions / 改善提案 (0.5 page)

  • How could you improve this model? | このモデルをどのように改善できるか？
  • What changes would you make to training data? | 訓練データにどのような変更を加えるか？
  • What features would you add? | どの特徴量を追加するか？
  • Realistic, specific suggestions | 現実的で具体的な提案

Format / 形式:

  • Section in main report document | メインレポート文書のセクション
  • 2-3 pages total | 合計2-3ページ
  • Use clear section headings | 明確なセクション見出しを使用
  • Be honest and thoughtful - there are no "wrong" answers for analysis | 正直で思慮深く - 分析に「間違った」答えはありません

Scoring (20 points) / 採点（20点）:

  • Error analysis depth: 6 points
  • Strengths identification: 3 points
  • Limitations and ethics discussion: 6 points
  • Improvement suggestions: 5 points

Tips for Success / 成功のためのヒント

Getting Started / 始め方

  • ✅ Work with the bot | ボットと作業する
    ML-101 Bot will guide you step-by-step through data collection, predictions, and metrics
  • ✅ Test on truly NEW companies | 本当に新しい企業でテストする
    Double-check that test companies weren't in your training data
  • ✅ Show your calculations | 計算を示す
    Don't just write "70% accuracy" - show how you got that number
  • ✅ Be honest in analysis | 分析において正直である
    It's okay if your model isn't perfect - critical analysis is what matters

Common Mistakes to Avoid / 避けるべきよくある間違い

  • ❌ Testing on training data
    This defeats the entire purpose of evaluation!
  • ❌ Incomplete confusion matrix
    Must include all four values: TP, TN, FP, FN
  • ❌ Calculations without interpretation
    Explain what the numbers MEAN, not just what they are
  • ❌ Shallow analysis
    "The model is good" is not sufficient. Explain WHY and analyze specific cases

Frequently Asked Questions / よくある質問

Q: What if my model accuracy is very low (like 50-60%)?

Q: モデルの精度が非常に低い場合（50-60%のような）どうなりますか？

A: That's okay! This is a learning project. What matters is your critical analysis of WHY it's low and how you'd improve it. Low accuracy with good analysis is better than high accuracy with no understanding.

A: それで大丈夫です！これは学習プロジェクトです。重要なのは、なぜ低いかの批判的分析とどのように改善するかです。理解のない高精度よりも、良い分析を伴う低精度の方が良いです。

Q: Can I use the same test data as my classmates?

Q: クラスメートと同じテストデータを使用できますか？

A: No. Each student must collect their own unique test data. You can use the same sources (mynavi.jp), but different specific companies.

A: いいえ。各学生は独自のテストデータを収集する必要があります。同じソース（mynavi.jp）を使用できますが、異なる特定の企業です。

Q: My model predicted everything as "Not Interested" - is that wrong?

Q: モデルがすべてを「興味なし」と予測しました - それは間違っていますか？

A: This is a common issue! It's not "wrong" for the assignment - it's an important finding to analyze. In your critical analysis, discuss WHY this happened (class imbalance? Need more diverse training data?) and how to fix it.

A: これは一般的な問題です！課題にとって「間違っている」わけではありません - それは分析すべき重要な発見です。批判的分析で、なぜこれが起こったか（クラスの不均衡？より多様な訓練データが必要？）とそれを修正する方法を議論してください。
