Why Model Evaluation Matters

The Critical Question

You've spent weeks building an ML system, but can you trust it?

Without evaluation, you have no idea if your model:

  • Makes good predictions
  • Generalizes to new data
  • Is better than random guessing
  • Has learned meaningful patterns

Model evaluation answers: "Does this system work, and how well?"

重要な質問

何週間もMLシステムの構築に費やしましたが、それを信頼できますか？

評価がなければ、モデルが以下かどうかわかりません：

  • 良い予測をする
  • 新しいデータに一般化する
  • ランダムな推測よりも優れている
  • 意味のあるパターンを学習した

モデル評価が答えます： 「このシステムは機能し、どれくらいうまく機能するか？」

Training Data vs Test Data

A Fundamental Concept in ML

Critical Rule / 重要なルール

NEVER evaluate your model on the same data it was trained on!

モデルが訓練されたのと同じデータでモデルを評価しないでください！

Why This Rule Matters / なぜこのルールが重要か

The Memorization Problem

Imagine studying for an exam by memorizing answers to specific questions. On the exact same test, you'd score 100%. But on a different test with new questions, you might fail completely.

ML models can do the same thing! They might "memorize" training data without learning general patterns.

Example: Your model might remember "Company A is interesting" without learning WHY (tech focus, startup culture, etc.).

暗記の問題

特定の質問への答えを暗記して試験勉強をすると想像してください。全く同じテストでは100点を取れるでしょう。しかし、新しい質問がある別のテストでは完全に失敗するかもしれません。

MLモデルも同じことができます！ 一般的なパターンを学習せずに訓練データを「暗記」するかもしれません。

例： モデルは、なぜか（技術志向、スタートアップ文化など）を学習せずに「企業Aは興味深い」と覚えているかもしれません。

Key Evaluation Metrics

1. Accuracy / 精度

Accuracy is the simplest and most intuitive metric. It answers: "What percentage of predictions were correct?"

精度は最もシンプルで直感的なメトリクスです。「予測の何パーセントが正しかったか？」に答えます。

Accuracy = (Number of Correct Predictions) / (Total Number of Predictions) 精度 = (正しい予測の数) / (予測の総数) Example / 例: Test on 10 companies / 10社でテスト: - 7 predicted correctly / 7つが正しく予測された - 3 predicted incorrectly / 3つが誤って予測された Accuracy = 7/10 = 0.70 = 70% 精度 = 7/10 = 0.70 = 70%

Interpreting Accuracy

For small datasets (10 test companies):

  • 50%: Random guessing (coin flip)
  • 60-70%: Model is learning something, but room for improvement
  • 70-80%: Good performance!
  • 80-90%: Excellent performance
  • 90-100%: Suspiciously high - might be overfitting

精度の解釈

小さなデータセット（10のテスト企業）の場合：

  • 50%: ランダムな推測（コイン投げ）
  • 60-70%: モデルは何かを学習しているが、改善の余地あり
  • 70-80%: 良いパフォーマンス！
  • 80-90%: 優れたパフォーマンス
  • 90-100%: 疑わしく高い - 過学習の可能性

2. Confusion Matrix / 混同行列

A confusion matrix provides a detailed breakdown of correct and incorrect predictions. It's called "confusion" matrix because it shows where the model gets "confused."

混同行列は、正しい予測と誤った予測の詳細な内訳を提供します。モデルがどこで「混乱」するかを示すため「混同」行列と呼ばれます。

PREDICTED / 予測 Not Interested | Interested 興味なし | 興味あり -------------------------------- ACTUAL Not | TN | FP 実際 興味 | 真陰性 | 偽陽性 なし | (Correct ✅) | (Type 1 Error ❌) -------------------------------- 興味 | FN | TP あり | 偽陰性 | 真陽性 | (Type 2 Error ❌)| (Correct ✅)

Four Outcomes Explained / 4つの結果の説明

True Positive (TP) / 真陽性:

  • Model predicted: "Interested"
  • Your actual label: "Interested"
  • Result: ✅ Correct match!
  • Example: Model correctly identified a tech startup you like

True Negative (TN) / 真陰性:

  • Model predicted: "Not Interested"
  • Your actual label: "Not Interested"
  • Result: ✅ Correct rejection!
  • Example: Model correctly filtered out a company you wouldn't want

False Positive (FP) / 偽陽性:

  • モデルの予測: 「興味あり」
  • 実際のラベル: 「興味なし」
  • 結果: ❌ タイプ1エラー
  • 例: モデルは実際には興味のない企業を推薦した

False Negative (FN) / 偽陰性:

  • モデルの予測: 「興味なし」
  • 実際のラベル: 「興味あり」
  • 結果: ❌ タイプ2エラー（機会損失！）
  • 例: モデルは素晴らしい企業の機会を逃した

Example Confusion Matrix / 混同行列の例

Test Results on 10 Companies / 10社のテスト結果: Predicted: Not | Predicted: Interested 予測: 興味なし | 予測: 興味あり ---------------------------------- Actual: Not | 4 (TN) | 1 (FP) 実際: 興味なし | | ---------------------------------- Actual: Yes | 2 (FN) | 3 (TP) 実際: 興味あり | | Interpretation / 解釈: - True Positives (TP): 3 companies correctly identified as interesting 真陽性: 興味深いと正しく識別された3社 - True Negatives (TN): 4 companies correctly identified as not interesting 真陰性: 興味深くないと正しく識別された4社 - False Positives (FP): 1 company wrongly recommended 偽陽性: 誤って推薦された1社 - False Negatives (FN): 2 great companies missed! 偽陰性: 逃した素晴らしい2社！ Total Correct = TP + TN = 3 + 4 = 7 Accuracy = 7/10 = 70%

3. Precision / 精密度（適合率）

Precision answers: "Of the companies my model said I'd be interested in, how many was I actually interested in?"

精密度が答える：「モデルが興味があると言った企業のうち、実際に興味があったのは何社か？」

Precision = TP / (TP + FP) 精密度 = 真陽性 / (真陽性 + 偽陽性) Using the example above / 上記の例を使用: Precision = 3 / (3 + 1) = 3/4 = 0.75 = 75% Interpretation / 解釈: Of the 4 companies model recommended, 3 were actually good matches (75%) モデルが推薦した4社のうち、3社が実際に良いマッチでした（75%）

When Precision Matters

High precision is important when:

  • False positives are costly
  • You only want to see companies you'll definitely like
  • You have limited time to review recommendations

Example: Job search - you don't want to waste time applying to companies you're not interested in.

精密度が重要な場合

高い精密度が重要な場合：

  • 偽陽性のコストが高い
  • 確実に気に入る企業のみを見たい
  • 推薦をレビューする時間が限られている

例： 就職活動 - 興味のない企業に応募するのに時間を無駄にしたくない。

4. Recall / 再現率

Recall answers: "Of all the companies I'm actually interested in, how many did my model find?"

再現率が答える：「実際に興味がある全ての企業のうち、モデルは何社見つけたか？」

Recall = TP / (TP + FN) 再現率 = 真陽性 / (真陽性 + 偽陰性) Using the example above / 上記の例を使用: Recall = 3 / (3 + 2) = 3/5 = 0.60 = 60% Interpretation / 解釈: Of the 5 companies you're actually interested in, model found 3 (60%) 実際に興味がある5社のうち、モデルは3社を見つけた（60%） It missed 2 good opportunities! / 2つの良い機会を逃した！

When Recall Matters

High recall is important when:

  • False negatives are costly
  • You don't want to miss good opportunities
  • You're willing to review more candidates to find all good matches

Example: Medical diagnosis - you don't want to miss detecting a disease.

再現率が重要な場合

高い再現率が重要な場合：

  • 偽陰性のコストが高い
  • 良い機会を逃したくない
  • すべての良いマッチを見つけるためにより多くの候補をレビューする意欲がある

例： 医療診断 - 病気の検出を逃したくない。

The Precision-Recall Trade-off / 精密度-再現率のトレードオフ

You Can't Maximize Both! / 両方を最大化できない！

English: There's often a trade-off between precision and recall. Improving one typically reduces the other.

日本語: 精密度と再現率の間にはトレードオフがあることが多いです。一方を改善すると通常他方が減少します。

  • High Precision, Lower Recall: Strict model - only recommends when very confident. Fewer false positives, but misses opportunities.
    高精密度、低再現率： 厳格なモデル - 非常に確信がある場合のみ推薦。偽陽性は少ないが、機会を逃す。
  • High Recall, Lower Precision: Generous model - recommends broadly. Finds all opportunities, but includes false positives.
    高再現率、低精密度： 寛大なモデル - 広範囲に推薦。すべての機会を見つけるが、偽陽性を含む。

Making Predictions on New Data

The Testing Process / テストプロセス

Step-by-Step

  1. Collect NEW test companies
    Find 5-10 companies NOT in your training data
  2. Prepare test data
    Apply same feature engineering as training data
  3. Make predictions
    Use your Week 11 model to predict "Interested" or "Not"
  4. Compare to truth
    Check predictions against your actual interest
  5. Calculate metrics
    Compute accuracy, confusion matrix, precision, recall

ステップバイステップ

  1. 新しいテスト企業を収集
    訓練データにない5-10社を見つける
  2. テストデータを準備
    訓練データと同じ特徴量エンジニアリングを適用
  3. 予測を行う
    第11週のモデルを使用して「興味あり」または「なし」を予測
  4. 真実と比較
    予測を実際の興味と照合
  5. メトリクスを計算
    精度、混同行列、精密度、再現率を計算

Important: Feature Consistency / 重要：特徴量の一貫性

Your test data MUST have the exact same features as training data!

テストデータは訓練データと全く同じ特徴量を持つ必要があります！

If your training data had:

  • Company name, industry, location, description
  • Keywords: "tech", "startup", "AI", "remote"
  • One-hot encoding for industry categories

Your test data needs THE SAME features in THE SAME format!

Critical Analysis of Model Performance

Beyond the Numbers / 数字を超えて

Evaluation isn't just about calculating metrics. The most important part is understanding WHY the model succeeds or fails.

評価はメトリクスを計算するだけではありません。最も重要な部分は、モデルがなぜ成功または失敗するかを理解することです。

Questions to Ask / 尋ねるべき質問

Analyzing Errors

  • Which companies were misclassified?
    Look at specific examples of failures
  • Why did the model fail?
    Missing features? Unclear patterns? Borderline cases?
  • Are errors systematic?
    Does model consistently miss certain types of companies?

Understanding Successes

  • What did the model learn correctly?
    Which patterns are most predictive?
  • Which features matter most?
    Keywords? Industry? Location?

エラーの分析

  • どの企業が誤分類されたか？
    失敗の具体的な例を見る
  • なぜモデルは失敗したか？
    特徴量の欠落？不明確なパターン？境界線ケース？
  • エラーは体系的か？
    モデルは特定のタイプの企業を一貫して見逃すか？

成功の理解

  • モデルは何を正しく学習したか？
    どのパターンが最も予測的か？
  • どの特徴量が最も重要か？
    キーワード？業界？場所？

Ethical Considerations / 倫理的考慮事項

Critical Thinking About AI / AIに関する批判的思考

As ML systems become more powerful, we must think critically about their limitations and impacts.

MLシステムがより強力になるにつれて、その限界と影響について批判的に考える必要があります。

Important Questions

  • Bias: Did my training labels introduce bias?
    Example: Only labeled tech companies as interesting → Model won't recommend non-tech
  • Transparency: Can I explain WHY the model made a prediction?
    Black box models vs interpretable models
  • Fairness: What if this was used for automated hiring?
    Could errors discriminate against certain candidates?
  • Trust: Would I actually use this for real job decisions?
    What level of accuracy would I need to trust it?

重要な質問

  • バイアス: 訓練ラベルがバイアスを導入したか？
    例：技術企業のみを興味深いとラベル付け → モデルは非技術企業を推薦しない
  • 透明性: モデルがなぜ予測を行ったかを説明できるか？
    ブラックボックスモデル vs 解釈可能なモデル
  • 公平性: これが自動採用に使用されたらどうなるか？
    エラーが特定の候補者を差別する可能性はあるか？
  • 信頼: 実際の仕事の決定にこれを使用するか？
    それを信頼するためにどのレベルの精度が必要か？

評価の目的は単にスコアを取得することではなく、モデルをより良くする方法を理解することです。

Key Takeaways

Remember / 覚えておいてください

  • ✅ Always test on NEW data the model hasn't seen / モデルが見ていない新しいデータで常にテスト
  • ✅ Accuracy is simple but doesn't tell the whole story / 精度はシンプルだが全体像は語らない
  • ✅ Confusion matrix shows where model succeeds and fails / 混同行列はモデルが成功および失敗する場所を示す
  • ✅ Precision and recall measure different aspects of performance / 精密度と再現率はパフォーマンスの異なる側面を測定
  • ✅ Critical analysis is more important than the numbers / 批判的分析は数字よりも重要
  • ✅ ML systems have limitations - think ethically about their use / MLシステムには限界があります - その使用について倫理的に考える
