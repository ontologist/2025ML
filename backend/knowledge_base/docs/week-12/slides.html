Training Data vs Test Data

訓練データ vs テストデータ

Training Data

訓練データ

What it is: The 20-30 companies you already labeled and used to train your model

それは何か: すでにラベル付けしてモデルの訓練に使用した20-30社

Purpose: Teaching the model your preferences

目的: モデルにあなたの好みを教える

Risk: Model might just "memorize" these companies

リスク: モデルがこれらの企業を「暗記」するだけかもしれない

Test Data

テストデータ

What it is: NEW companies the model has NEVER seen before

それは何か: モデルが以前に見たことがない新しい企業

Purpose: Evaluating if model can generalize to new data

目的: モデルが新しいデータに一般化できるかを評価する

Why Important: True test of model's learning ability

なぜ重要か: モデルの学習能力の真のテスト

Critical Rule: NEVER test on data the model was trained on!

重要なルール: モデルが訓練されたデータでテストしないこと！

Key Evaluation Metrics

主要な評価メトリクス

1. Accuracy / 精度

Accuracy = (Correct Predictions) / (Total Predictions) 精度 = (正しい予測) / (総予測数) Example: 7 correct out of 10 companies = 70% accuracy 例: 10社のうち7社正しい = 70%の精度

Simple interpretation: "The model got 7 out of 10 right"

シンプルな解釈: 「モデルは10のうち7を正しく予測した」

2. Confusion Matrix / 混同行列

Predicted: Not Interested | Predicted: Interested 予測: 興味なし | 予測: 興味あり Actual: Not True Negatives (TN) | False Positives (FP) Interested 真陰性 | 偽陽性 実際: 興味なし Correct rejections | Wrong "interested" Actual: False Negatives (FN) | True Positives (TP) Interested 偽陰性 | 真陽性 実際: 興味あり Missed opportunities | Correct matches!

Understanding Confusion Matrix

混同行列の理解

Four Outcomes:

  • True Positive (TP):
    Model said "Interested" and you agree → ✅ Correct!
  • True Negative (TN):
    Model said "Not Interested" and you agree → ✅ Correct!
  • False Positive (FP):
    Model said "Interested" but you're actually not → ❌ Type 1 Error
  • False Negative (FN):
    Model said "Not Interested" but you actually are → ❌ Type 2 Error (Missed opportunity!)

4つの結果:

  • 真陽性 (TP):
    モデルが「興味あり」と言い、あなたも同意 → ✅ 正しい！
  • 真陰性 (TN):
    モデルが「興味なし」と言い、あなたも同意 → ✅ 正しい！
  • 偽陽性 (FP):
    モデルが「興味あり」と言ったが実際は興味なし → ❌ タイプ1エラー
  • 偽陰性 (FN):
    モデルが「興味なし」と言ったが実際は興味あり → ❌ タイプ2エラー（機会損失！）

Advanced Evaluation Metrics

高度な評価メトリクス

Precision / 精密度（適合率）

Precision = TP / (TP + FP) 精密度 = 真陽性 / (真陽性 + 偽陽性) Question: Of companies model said "Interested", how many actually were? 質問: モデルが「興味あり」と言った企業のうち、実際に何社がそうでしたか？

Recall / 再現率

Recall = TP / (TP + FN) 再現率 = 真陽性 / (真陽性 + 偽陰性) Question: Of companies you're ACTUALLY interested in, how many did model find? 質問: 実際に興味がある企業のうち、モデルは何社見つけましたか？

Trade-off: High precision OR high recall? You often can't have both perfectly!

トレードオフ: 高い精密度か高い再現率か？両方を完璧には持てないことが多い！

Interpreting Your Model's Results

モデルの結果の解釈

What's a "Good" Accuracy? / 「良い」精度とは？

For small datasets (20-30 companies):

  • 60-70% → Okay, room for improvement
  • 70-80% → Good performance!
  • 80-90% → Excellent!
  • 90-100% → Suspicious... might be overfitting

Context matters:

  • 50% accuracy = random guessing (like flipping a coin)
  • Small datasets have more variability
  • Learning from mistakes is more important than perfect accuracy

小さなデータセット（20-30社）の場合:

  • 60-70% → まあまあ、改善の余地あり
  • 70-80% → 良いパフォーマンス！
  • 80-90% → 優秀！
  • 90-100% → 疑わしい...過学習の可能性

文脈が重要:

  • 50%の精度 = ランダムな推測（コインを投げるのと同じ）
  • 小さなデータセットはより変動性が高い
  • 間違いから学ぶことが完璧な精度より重要

Critical Thinking About AI Systems

AIシステムに関する批判的思考

Important Questions to Ask / 尋ねるべき重要な質問

  • Bias: Did my training labels introduce bias?
    Example: Only labeled tech companies as "interested"
  • Limitations: What can't this model do?
    Example: Can't understand company culture
  • Ethics: What if this was used for hiring decisions?
    Would errors be fair to job seekers?
  • Transparency: Can I explain WHY the model made a prediction?
    Black box vs interpretable models
  • バイアス: 訓練ラベルがバイアスを導入したか？
    例: 技術企業のみを「興味あり」とラベル付け
  • 限界: このモデルができないことは何か？
    例: 企業文化を理解できない
  • 倫理: これが採用決定に使用されたらどうなるか？
    エラーは求職者に公平か？
  • 透明性: モデルがなぜ予測を行ったかを説明できるか？
    ブラックボックス vs 解釈可能なモデル

Remember: ML is a tool, not magic. It has strengths AND weaknesses.

覚えておいて: MLはツールであり、魔法ではありません。強みと弱みがあります。
