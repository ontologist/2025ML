What is a Data Schema?

データスキーマとは？

Definition / 定義:

A data schema is the blueprint for organizing your data

データスキーマはデータを整理するための設計図です

What It Defines:

  • Field Names: What data you'll collect (e.g., "Company Name", "Industry")
  • Data Types: How to store each field (text, number, date)
  • Structure: How fields relate to each other
  • Requirements: Which fields are required vs optional

定義する内容:

  • フィールド名: 収集するデータ（例：「企業名」、「業界」）
  • データ型: 各フィールドの保存方法（テキスト、数値、日付）
  • 構造: フィールド間の関係
  • 要件: 必須フィールドとオプションフィールド

Why Schema Design Matters

なぜスキーマ設計が重要か

Good Schema Design Leads To:

良いスキーマ設計は以下につながる:

  • Consistent, usable data
  • Easier data collection
  • Better ML model performance
  • Clear project direction

一貫性のある使用可能なデータ、より簡単なデータ収集、より良いMLモデルパフォーマンス、明確なプロジェクト方向性

Poor Schema Design Leads To:

悪いスキーマ設計は以下につながる:

  • Messy, inconsistent data
  • Difficult analysis
  • Wasted time cleaning data
  • Poor model results

乱雑で一貫性のないデータ、困難な分析、データクリーニングに費やされる時間、悪いモデル結果

Choosing Data Sources

データソースの選択

Primary Sources (Main Data):

  • Job listing sites (approved in Week 4)
  • Company directories
  • Industry databases

Secondary Sources (Supplemental):

  • Company websites
  • Wikipedia or business profiles
  • LinkedIn company pages

主要ソース（メインデータ）:

  • 求人サイト（第4週で承認済み）
  • 企業ディレクトリ
  • 業界データベース

二次ソース（補完）:

  • 企業ウェブサイト
  • Wikipediaまたはビジネスプロフィール
  • LinkedIn企業ページ

Remember: Use only approved websites. Respect robots.txt and rate limits!

覚えておいてください: 承認されたウェブサイトのみを使用。robots.txtとレート制限を尊重！

Data Quality Principles

データ品質の原則

The 4 C's of Data Quality / データ品質の4つのC

1. Completeness / 完全性

Minimize missing fields

欠損フィールドを最小化

2. Consistency / 一貫性

Use same formats throughout

全体で同じフォーマットを使用

3. Correctness / 正確性

Verify information is accurate

情報が正確であることを確認

4. Clarity / 明確性

Data should be understandable

データは理解可能であるべき

Quality Check: Before finishing, calculate completeness percentage for each field

品質チェック: 終了前に各フィールドの完全性パーセンテージを計算

Common Challenges & Solutions

よくある課題と解決策

Challenge 1: Can't Find 20-30 Companies

課題1: 20-30社を見つけられない

Solution: Combine multiple approved sources. Use different search terms.

解決策: 複数の承認されたソースを組み合わせる。異なる検索用語を使用。

Challenge 2: Missing Data Fields

課題2: データフィールドの欠損

Solution: Focus on required fields first. Optional fields can be incomplete.

解決策: まず必須フィールドに焦点を当てる。オプションフィールドは不完全でも可。

Challenge 3: Inconsistent Formatting

課題3: 一貫性のないフォーマット

Solution: Create standardization rules early. Apply them to all entries.

解決策: 早期に標準化ルールを作成。すべてのエントリに適用。
