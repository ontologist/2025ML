Assignment Workflow / 課題ワークフロー

Follow these five steps to complete your data cleaning assignment:

データクリーニング課題を完了するには、次の5つのステップに従ってください：

Step 1: Data Quality Assessment (10 minutes)

ステップ1：データ品質評価（10分）

Tasks:

  1. Open Google Colab and create new notebook: ML101_Week7_DataCleaning
  2. Upload your Week 6 company dataset CSV file
  3. Load data using pandas: df = pd.read_csv('companies.csv')
  4. Run diagnostic commands:
import pandas as pd

# Load dataset
df = pd.read_csv('companies.csv')

# View first rows
df.head()

# Check data types and null values
df.info()

# Get statistics
df.describe()

# Count duplicates
print(f"Duplicates found: {df.duplicated().sum()}")

Document identified issues: duplicates, missing values, inconsistent formatting, data type errors

Step 2: Removing Duplicates and Standardizing Text (12 minutes)

ステップ2：重複の削除とテキストの標準化（12分）

Cleaning operations:

# Remove exact duplicates
print(f"Duplicates found: {df.duplicated().sum()}")
df = df.drop_duplicates()

# Standardize company names
df['company_name'] = df['company_name'].str.strip()
df['company_name'] = df['company_name'].str.title()

# Standardize industry categories
df['industry'] = df['industry'].str.strip().str.lower()
df['industry'] = df['industry'].replace({
    'tech': 'technology',
    'it': 'technology'
})

# Check unique values
print(df['industry'].unique())

Step 3: Handling Data Type Issues (10 minutes)

ステップ3：データ型問題の処理（10分）

Type conversions:

# Check current data types
print(df.dtypes)

# Convert employee_count to numeric
df['employee_count'] = pd.to_numeric(df['employee_count'], errors='coerce')

# Convert founding_year to integer (Int64 allows NaN)
df['founding_year'] = df['founding_year'].astype('Int64')

# Remove currency symbols from revenue
df['revenue'] = df['revenue'].str.replace('[$¥,]', '', regex=True)
df['revenue'] = pd.to_numeric(df['revenue'], errors='coerce')

# Verify conversions
print(df.dtypes)

Step 4: Consistency Checks and Corrections (13 minutes)

ステップ4：一貫性チェックと修正（13分）

Validation operations:

# Check for logical inconsistencies
current_year = 2025
invalid_years = df[df['founding_year'] > current_year]
print(f"Invalid founding years: {len(invalid_years)}")

# Standardize location formats
df['location'] = df['location'].str.strip().str.title()

# Check for impossible values
print(f"Negative employee counts: {(df['employee_count'] < 0).sum()}")

# Mark suspicious entries
df['needs_review'] = False
df.loc[df['employee_count'] < 0, 'needs_review'] = True
df.loc[df['founding_year'] > current_year, 'needs_review'] = True

Step 5: Final Validation and Documentation (15 minutes)

ステップ5：最終検証と文書化（15分）

Final checks and export:

# Final quality check
print("Dataset shape:", df.shape)
print("\nMissing values:")
print(df.isnull().sum())
print("\nData types:")
print(df.dtypes)
print("\nDuplicate check:", df.duplicated().sum())

# Export cleaned dataset
df.to_csv('companies_cleaned.csv', index=False)

Document in markdown cells:

  • Issues identified
  • Cleaning operations performed
  • Impact of each operation
  • Remaining limitations

Tips for Success / 成功のためのヒント

Getting Started / 始め方

  • ✅ Start with data assessment | データ評価から始める
    Don't clean randomly - first understand what issues exist | ランダムにクリーニングしない - 最初にどのような問題が存在するかを理解する
  • ✅ Work systematically | 体系的に作業する
    Follow the five-step workflow: assess, duplicates, types, consistency, validate | 5ステップワークフローに従う：評価、重複、型、一貫性、検証
  • ✅ Save your work frequently | 頻繁に作業を保存する
    Colab notebooks can disconnect - download regularly | Colabノートブックは切断される可能性がある - 定期的にダウンロード
  • ✅ Test code incrementally | コードを段階的にテストする
    Run each cleaning operation separately and check results with df.head() | 各クリーニング操作を個別に実行し、df.head()で結果を確認

Common Mistakes to Avoid / 避けるべきよくある間違い

  • ❌ Forgetting to reassign DataFrame
    Use df = df.drop_duplicates() not just df.drop_duplicates()
  • ❌ Not checking column names
    Use df.columns to see exact names - they're case-sensitive!
  • ❌ Over-cleaning
    Don't delete valuable data unnecessarily - some missing values are okay
  • ❌ Poor documentation
    Explain WHY you made decisions, not just WHAT you did
  • ❌ Not validating results
    Always check that your cleaning operations actually worked

Common Issues & Solutions / 一般的な問題と解決策

Problem: Can't upload CSV file to Colab

問題：ColabにCSVファイルをアップロードできない

Solution: Click folder icon on left sidebar, then upload icon. Or mount Google Drive and access files from there. Make sure file name matches what you use in code.

解決策: 左サイドバーのフォルダアイコンをクリックし、次にアップロードアイコンをクリック。またはGoogle Driveをマウントしてそこからファイルにアクセス。ファイル名がコードで使用するものと一致することを確認。

Problem: Getting KeyError when accessing columns

問題：列にアクセスするとKeyErrorが発生する

Solution: Check exact column names with df.columns. Column names are case-sensitive. Make sure you're using the exact name including any spaces.

解決策: df.columnsで正確な列名を確認。列名は大文字小文字を区別します。スペースを含む正確な名前を使用していることを確認。

Problem: Data type conversion creates NaN values

問題：データ型変換がNaN値を作成する

Solution: This is normal when there's invalid data. Use errors='coerce' parameter. Check which values became NaN: df[df['column'].isna()]. Decide if you should fix source data or handle these as missing values.

解決策: これは無効なデータがある場合は正常です。errors='coerce'パラメータを使用。どの値がNaNになったかをチェック：df[df['column'].isna()]。ソースデータを修正すべきか、これらを欠損値として処理すべきかを決定。

Problem: Code runs but doesn't change the data

問題：コードは実行されるがデータを変更しない

Solution: Remember to reassign to the DataFrame: df = df.drop_duplicates() not just df.drop_duplicates(). Some pandas operations return a new DataFrame. Check with df.head() after each operation.

解決策: DataFrameに再割り当てすることを忘れないでください：df.drop_duplicates()だけでなくdf = df.drop_duplicates()。一部のpandas操作は新しいDataFrameを返します。各操作後にdf.head()でチェック。
