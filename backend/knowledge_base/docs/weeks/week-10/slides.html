What is Classification?

分類とは？

Definition / 定義:

Classification = Putting things into categories

分類 = 物をカテゴリに入れること

Everyday Examples

日常の例

  • Email: Spam vs Not Spam
    メール: スパム vs 非スパム
  • Fruit: Apple vs Orange
    果物: リンゴ vs オレンジ
  • Weather: Rainy vs Sunny
    天気: 雨 vs 晴れ

ML Classification

ML分類

  • Computer learns from examples
    コンピュータが例から学習
  • Finds patterns in data
    データのパターンを見つける
  • Makes predictions on new data
    新しいデータで予測を行う

Types of Classification

分類のタイプ

Binary Classification / 二項分類

Two categories only / 2つのカテゴリのみ

Examples / 例: • Email: Spam (1) or Not Spam (0) メール: スパム (1) または非スパム (0) • Medical: Disease (1) or Healthy (0) 医療: 病気 (1) または健康 (0) • Our Project: Interested (1) or Not Interested (0) 私たちのプロジェクト: 興味あり (1) または興味なし (0)

Multi-Class Classification / 多クラス分類

Three or more categories / 3つ以上のカテゴリ

Examples / 例: • Fruit: Apple, Orange, Banana 果物: リンゴ、オレンジ、バナナ • Handwriting: Digits 0-9 手書き: 数字0-9 • Sentiment: Positive, Neutral, Negative 感情: ポジティブ、中立、ネガティブ

Supervised Learning

教師あり学習

Key Concept / 重要コンセプト

"Learning from labeled examples"

「ラベル付きの例から学習」

How It Works / どのように機能するか

Step 1: Provide LABELED data ステップ1: ラベル付きデータを提供 → Show computer examples with correct answers 正しい答えを含む例をコンピュータに示す Step 2: Computer LEARNS patterns ステップ2: コンピュータがパターンを学習 → Finds what features predict each label 各ラベルを予測する特徴量を見つける Step 3: Computer PREDICTS on new data ステップ3: コンピュータが新しいデータで予測 → Uses learned patterns to classify unseen examples 学習したパターンを使用して未見の例を分類

Features vs Labels

特徴量 vs ラベル

Features (X)

特徴量 (X)

Input / 入力

  • Characteristics that describe something
    何かを記述する特性
  • What the model learns FROM
    モデルが学習する元
  • Example: Company size, industry, location
    例: 企業規模、業界、所在地

Labels (y)

ラベル (y)

Output / 出力

  • The category we want to predict
    予測したいカテゴリ
  • What the model learns TO predict
    モデルが予測することを学習するもの
  • Example: Interested (1) or Not Interested (0)
    例: 興味あり (1) または興味なし (0)

The ML Goal / MLの目標

Learn the relationship: Features → Label

関係を学習: 特徴量 → ラベル

Our Project: Company Classification

私たちのプロジェクト: 企業分類

The Classification Task / 分類タスク

Goal: Classify companies as "Interested" or "Not Interested"

目標: 企業を「興味あり」または「興味なし」に分類

Features (What we know about companies) / 特徴量（企業について知っていること）

  • Industry type / 業界タイプ
  • Company size / 企業規模
  • Location / 所在地
  • Keywords in description / 説明のキーワード
  • All the features you engineered in Week 9! / 第9週でエンジニアリングしたすべての特徴量！

Labels (Your decision) / ラベル（あなたの決定）

  • 1 = Interested in this company / この企業に興味がある
  • 0 = Not interested in this company / この企業に興味がない

Training Data vs Test Data

訓練データ vs テストデータ

Training Data

訓練データ

Used to LEARN

学習に使用

  • Labeled examples / ラベル付き例
  • Model learns patterns / モデルがパターンを学習
  • Typically 70-80% of data / 通常データの70-80%

Test Data

テストデータ

Used to EVALUATE

評価に使用

  • Hidden during training / 訓練中は隠される
  • Tests model accuracy / モデルの精度をテスト
  • Typically 20-30% of data / 通常データの20-30%

Important! / 重要！

Never train on test data - that's cheating!

テストデータで訓練してはいけません - それはズルです！

Why Label Quality Matters

ラベル品質が重要な理由

Garbage In, Garbage Out

ゴミを入れれば、ゴミが出る

Your model is only as good as your labels!

モデルはラベルと同じくらい良い程度です！

Good Labels / 良いラベル

  • Consistent: Similar companies get similar labels
    一貫性: 類似の企業が類似のラベルを取得
  • Thoughtful: Based on clear criteria
    思慮深い: 明確な基準に基づく
  • Honest: Reflect YOUR true preferences
    正直: あなたの真の好みを反映
  • Balanced: Not all one category
    バランス: すべて1つのカテゴリではない

Bad Labels / 悪いラベル

  • Random or careless / ランダムまたは不注意
  • Inconsistent decisions / 不整合な決定
  • Extremely imbalanced (90% one class) / 極端に不均衡（90%が1つのクラス）
