What are No-Code ML Platforms?

ノーコードMLプラットフォームとは？

Definition / 定義:

No-Code ML = Machine learning without programming

ノーコードML = プログラミングなしの機械学習

Traditional ML

従来のML

  • Write Python code
    Pythonコードを書く
  • Manage libraries
    ライブラリを管理
  • Debug errors
    エラーをデバッグ
  • Configure parameters
    パラメータを設定

No-Code ML

ノーコードML

    データをアップロード (CSV)
  • Click buttons
    ボタンをクリック
  • Platform handles code
    プラットフォームがコードを処理
  • Visual interface
    ビジュアルインターフェース

Why Use No-Code Platforms?

なぜノーコードプラットフォームを使用するのか？

Benefits / 利点

  • Accessibility: No programming knowledge required
  • Speed: Train models in minutes, not hours
  • Focus on concepts: Learn ML principles without coding distractions
  • Professional tools: Same algorithms as code-based ML
  • Real results: Actually works for real problems
  • アクセシビリティ: プログラミング知識不要
  • スピード: 数時間ではなく数分でモデルを訓練
  • コンセプトに集中: コーディングの煩わしさなしにML原則を学ぶ
  • プロフェッショナルツール: コードベースのMLと同じアルゴリズム
  • 実際の結果: 実際の問題に対して機能

The Model Training Process

モデル訓練プロセス

What Happens During Training? / 訓練中に何が起こるか？

STEP 1: Data Upload / データアップロード → Platform reads your labeled CSV file プラットフォームがラベル付きCSVファイルを読み取る STEP 2: Configuration / 設定 → Identify target variable (label column) ターゲット変数を特定（ラベル列） → Identify features (input columns) 特徴量を特定（入力列） STEP 3: Model Selection / モデル選択 → Platform tries different ML algorithms プラットフォームが異なるMLアルゴリズムを試す → Finds which works best for your data データに最適なものを見つける STEP 4: Training / 訓練 → Model learns patterns from labeled data モデルがラベル付きデータからパターンを学習 → Adjusts internal parameters 内部パラメータを調整 STEP 5: Validation / 検証 → Tests model on held-out data 保持されたデータでモデルをテスト → Reports accuracy and performance 精度とパフォーマンスを報告

Understanding Model Selection

モデル選択の理解

Common Classification Algorithms / 一般的な分類アルゴリズム

  • Logistic Regression / ロジスティック回帰
    Simple, fast, works well for small datasets
    シンプル、高速、小さなデータセットでうまく機能
  • Decision Trees / 決定木
    Easy to understand, visual decision rules
    理解しやすい、視覚的な決定ルール
  • Random Forest / ランダムフォレスト
    Multiple decision trees, more accurate
    複数の決定木、より正確
  • Neural Networks / ニューラルネットワーク
    Complex patterns, needs more data
    複雑なパターン、より多くのデータが必要

Good news: No-code platforms automatically choose the best algorithm!

良いニュース: ノーコードプラットフォームが自動的に最適なアルゴリズムを選択します！

Training vs Testing Data

訓練データ vs テストデータ

Training Split

訓練分割

70-80% of data

  • Model learns patterns
    モデルがパターンを学習
  • Sees features and labels
    特徴量とラベルを見る
  • Adjusts to minimize errors
    エラーを最小限に抑えるように調整

Testing Split

テスト分割

20-30% of data

  • Model evaluation
    モデル評価
  • Hidden during training
    訓練中は隠される
  • Measures true performance
    真のパフォーマンスを測定

Platform automatically splits your data!

プラットフォームがデータを自動的に分割します！

Understanding Training Metrics

訓練メトリクスの理解

Accuracy / 精度

What percentage of predictions were correct?

予測のうち何パーセントが正しかったか？

Accuracy = Correct Predictions / Total Predictions 精度 = 正しい予測 / 総予測数 Example: 85% accuracy means 85 out of 100 predictions were right 例: 85%の精度は、100の予測のうち85が正しかったことを意味

What's a Good Accuracy? / 良い精度とは？

  • 50% = Random guessing (for binary classification)
    ランダム推測（二項分類の場合）
  • 60-70% = Model is learning something!
    モデルが何かを学習している！
  • 80-90% = Good performance
    良好なパフォーマンス
  • 95%+ = Excellent (but check for overfitting)
    優秀（ただし過学習をチェック）

What to Expect from Your Model

モデルに何を期待するか

Realistic Expectations / 現実的な期待

With ~25 companies in your dataset, expect 60-80% accuracy

データセットに約25社の場合、60-80%の精度を期待

Why Not Perfect? / なぜ完璧ではないのか？

  • Small dataset: Limited examples to learn from
    小さなデータセット: 学習する例が限られている
  • Complex task: Human preferences are nuanced
    複雑なタスク: 人間の好みは微妙
  • Feature limitations: Some important factors might not be captured
    特徴量の制限: いくつかの重要な要因がキャプチャされていない可能性
