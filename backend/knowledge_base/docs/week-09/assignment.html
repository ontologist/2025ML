Activity 2: Creating Categorical Features (12 minutes)

アクティビティ2：カテゴリカル特徴量の作成（12分）

What to Do:

  1. One-hot encode the industry column
    業界列をワンホットエンコード
  2. One-hot encode the location column (if using)
    所在地列をワンホットエンコード（使用する場合）
  3. Create company size categories from employee count
    従業員数から企業規模カテゴリを作成
  4. One-hot encode the size categories
    サイズカテゴリをワンホットエンコード

Example Code / コード例:

# One-hot encode industry
industry_encoded = pd.get_dummies(df['industry'], prefix='industry')
df = pd.concat([df, industry_encoded], axis=1)

# Create size categories
def categorize_size(count):
    if pd.isna(count):
        return 'Unknown'
    elif count < 50:
        return 'Small'
    elif count < 500:
        return 'Medium'
    else:
        return 'Large'

df['company_size'] = df['employee_count'].apply(categorize_size)
size_encoded = pd.get_dummies(df['company_size'], prefix='size')
df = pd.concat([df, size_encoded], axis=1)

Expected Output:

  • Dataset with encoded categorical features
    エンコードされたカテゴリカル特徴量を含むデータセット

Activity 3: Text Feature Extraction - Keywords (15 minutes)

アクティビティ3：テキスト特徴量抽出 - キーワード（15分）

What to Do:

  1. Define 5-10 keywords relevant to your interests
    あなたの興味に関連する5-10のキーワードを定義
  2. Create binary features checking if keywords appear in descriptions
    キーワードが説明に現れるかをチェックするバイナリ特徴量を作成
  3. Create keyword count feature
    キーワードカウント特徴量を作成
  4. Create description length features
    説明の長さ特徴量を作成

Example Code / コード例:

# Define your interest keywords
keywords = ['innovation', 'technology', 'global', 'startup', 'research']

# Create binary feature for each keyword
for keyword in keywords:
    df[f'has_{keyword}'] = df['description'].str.lower().str.contains(
        keyword, na=False
    ).astype(int)

# Count total keywords present
df['keyword_count'] = df[[f'has_{keyword}' for keyword in keywords]].sum(axis=1)

# Description length
df['description_length'] = df['description'].str.len()
df['description_word_count'] = df['description'].str.split().str.len()

Expected Output:

  • Dataset with keyword-based text features
    キーワードベースのテキスト特徴量を含むデータセット

Activity 4: Advanced Text Features - TF-IDF (Optional) (13 minutes)

アクティビティ4：高度なテキスト特徴量 - TF-IDF（オプション）（13分）

What to Do:

  1. Learn about TF-IDF (Term Frequency-Inverse Document Frequency)
    TF-IDFについて学ぶ
  2. Apply TF-IDF to company descriptions
    企業説明にTF-IDFを適用
  3. Examine which words TF-IDF identified as important
    TF-IDFがどの単語を重要として特定したかを調査

Example Code / コード例:

from sklearn.feature_extraction.text import TfidfVectorizer

# Create TF-IDF features (limit to top 20 words)
tfidf = TfidfVectorizer(max_features=20, stop_words='english')
tfidf_matrix = tfidf.fit_transform(df['description'].fillna(''))

# Convert to DataFrame
tfidf_df = pd.DataFrame(
    tfidf_matrix.toarray(),
    columns=[f'tfidf_{word}' for word in tfidf.get_feature_names_out()]
)

# Add to main dataset
df = pd.concat([df.reset_index(drop=True), tfidf_df], axis=1)

# See which words are most important
print("Top TF-IDF features:", tfidf.get_feature_names_out())

注意: このアクティビティはオプションです。複雑すぎる場合は、スキップしてキーワード特徴量に焦点を当てることができます。

Activity 5: Feature Dataset Preparation and Validation (10 minutes)

アクティビティ5：特徴量データセット準備と検証（10分）

What to Do:

  1. Select final feature set for ML training
    ML訓練のための最終特徴量セットを選択
  2. Remove original text columns (keep only engineered features)
    元のテキスト列を削除（エンジニアリングされた特徴量のみを保持）
  3. Create clean feature dataset
    クリーンな特徴量データセットを作成
  4. Validate feature quality
    特徴量品質を検証
  5. Export feature dataset
    特徴量データセットをエクスポート
  6. Document feature engineering process
    特徴量エンジニアリングプロセスを文書化

Example Code / コード例:

# Identify feature columns
feature_columns = [col for col in df.columns if any([
    col.startswith('industry_'),
    col.startswith('location_'),
    col.startswith('size_'),
    col.startswith('has_'),
    col.startswith('tfidf_'),
    col == 'keyword_count',
    col == 'description_length',
    col == 'employee_count'
])]

# Create feature dataset (keep company_name for reference)
features_df = df[['company_name'] + feature_columns].copy()

print(f"Total features created: {len(feature_columns)}")
print(f"Feature columns: {feature_columns}")

# Validate
print("\nAll features numeric?", features_df[feature_columns].dtypes.apply(lambda x: x in ['int64', 'float64']).all())
print("Any missing values?", features_df[feature_columns].isnull().sum().sum())

# Export
features_df.to_csv('company_features.csv', index=False)

Feature Quality Checklist / 特徴量品質チェックリスト:

  • ✓ All features are numeric
  • ✓ No missing values in features
  • ✓ Reasonable number of features (15-50)
  • ✓ Features make sense for classification
